## 2. Source Code
The source code for this project is organized as follows:

- **Core Logic:** `index.php` (Login), `dashboard.php` (Main Hub), `includes/db.php` (Database Connection)
- **Features:** 
  - `transcribe.php` (Speech-to-Text)
  - `dictionary.php` (Real-time Lookup)
  - `focus.php` (Pomodoro Timer)
  - `read_aloud.php` (Text-to-Speech)
  - `export.php` (Data Export)
  - `profile.php` (User Settings)
- **APIs (JSON Endpoints):** `api/save_transcript.php`, `api/get_transcripts.php`, `api/dictionary_lookup.php`, `api/log_export.php`
- **Styling:** `css/style.css` (Glassmorphism & Light/Dark Themes)
- **Database:** `setup.sql` (Schema), `setup.php` (Installer)

## 3. Discussion (Experience Reflection)
**Goal:** The objective was to build a "Smart Study Assistant" that integrates accessibility features like speech-to-text and text-to-speech into a cohesive web application. I aimed to create a modern, "glassmorphism" interface that feels premium and distraction-free.

**Challenges:** The main challenge was integrating multiple asynchronous APIs. For instance, connecting the real-time Web Speech API for transcription with a backend PHP database required careful handling of JSON data streams. Additionally, implementing the "Instant Search" for the dictionary involved debouncing user input to prevent flooding the external API.

**Fascinating:** I found it fascinating how powerful modern browser APIs are. We didn't need complex third-party libraries for speech recognition or synthesisâ€”it's all built into the browser. Connecting this frontend capability with a robust MySQL backend created a seamless experience where data flows instantly between the user's voice and the database.
